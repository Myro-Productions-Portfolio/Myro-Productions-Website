# SEO Implementation Guide

This document outlines the SEO and metadata implementation for the Myro Productions website.

## What's Been Implemented

### 1. Root Layout Metadata (`app/layout.tsx`)
- **Title Template**: Pages will show as "Page Name | Myro Productions"
- **Default Title**: "Myro Productions | Rapid Prototyping & AI Development"
- **Description**: Optimized for search engines with key services
- **Keywords**: Targeted keywords for SEO
- **Open Graph**: Social media sharing metadata (Facebook, LinkedIn)
- **Twitter Cards**: Optimized Twitter sharing
- **Robots**: Configured for optimal crawling

### 2. JSON-LD Structured Data (`components/seo/JsonLd.tsx`)
Added four schema types for rich search results:
- **Organization Schema**: Company information
- **WebSite Schema**: Website metadata
- **Person Schema**: Nicolas Robert Myers profile
- **ProfessionalService Schema**: Service offerings

### 3. Sitemap (`app/sitemap.ts`)
Dynamic sitemap generation including:
- Homepage (priority 1.0)
- Services section (priority 0.9)
- Portfolio section (priority 0.8)
- About section (priority 0.7)
- Contact section (priority 0.9)

### 4. Robots.txt (`app/robots.ts`)
- Allows all crawlers
- Blocks API routes and Next.js internals
- References sitemap location

### 5. PWA Manifest (`public/manifest.json`)
Progressive Web App configuration:
- Theme colors matching design system
- App icons (requires creation)
- Standalone display mode

### 6. Dynamic Favicons
- **Standard Favicon** (`app/icon.tsx`): 32x32 dynamic favicon
- **Apple Touch Icon** (`app/apple-icon.tsx`): 180x180 for iOS
- **Open Graph Image** (`app/opengraph-image.tsx`): 1200x630 for social sharing

### 7. Environment Configuration
Created `.env.example` for:
- Site URL configuration
- Optional analytics
- Optional verification tokens

## Assets You Need to Create

### Required Images
1. **Icon Files** (for manifest.json):
   - `/public/icon-192.png` (192x192)
   - `/public/icon-512.png` (512x512)
   - These should be the Myro Productions logo

2. **Logo** (for JSON-LD):
   - `/public/logo.png` (recommended: 512x512 or larger)
   - Square format with transparent background

3. **Fallback Open Graph Image** (optional):
   - `/public/og-image.png` (1200x630)
   - If you want a static fallback instead of dynamic generation

### Icon Creation Tips
- Use the moss-900 (#1a2e1a) and moss-400 (#82f582) colors from your design system
- Keep it simple and recognizable at small sizes
- Export as PNG with transparent background
- Use online tools like Canva, Figma, or Adobe Express

## Configuration Steps

### 1. Set Environment Variables
Create a `.env.local` file (not tracked in git):

```bash
NEXT_PUBLIC_SITE_URL=https://myroproductions.com
```

For production, add this to your hosting platform (Vercel/Netlify).

### 2. Create Required Images
See "Assets You Need to Create" above.

### 3. Add Social Media Links
When you have social media profiles, update:
- `components/seo/JsonLd.tsx` - uncomment and add URLs in `sameAs` arrays
- Both Organization and Person schemas

### 4. Set Up Search Console Verification (After Deployment)

**Google Search Console:**
1. Go to https://search.google.com/search-console
2. Add your property (https://myroproductions.com)
3. Choose "HTML tag" verification method
4. Copy the verification token
5. Add to `app/layout.tsx` in the `verification.google` field
6. Redeploy

**Bing Webmaster Tools:**
1. Go to https://www.bing.com/webmasters
2. Add your site
3. Get verification token
4. Add to `app/layout.tsx` in the `verification.bing` field
5. Redeploy

### 5. Submit Sitemap
After deployment:
- Google: Submit `https://myroproductions.com/sitemap.xml` in Search Console
- Bing: Submit in Webmaster Tools

## How SEO Works in Next.js 15

### Metadata System
Next.js 15 uses the App Router's metadata API:
- Export `metadata` object from `layout.tsx` and `page.tsx`
- Metadata is automatically merged (page overrides layout)
- Title template allows consistent branding

### Dynamic Generation
These files are auto-generated by Next.js:
- `/sitemap.xml` from `app/sitemap.ts`
- `/robots.txt` from `app/robots.ts`
- `/icon` from `app/icon.tsx`
- `/apple-icon` from `app/apple-icon.tsx`
- `/opengraph-image` from `app/opengraph-image.tsx`

### Benefits
- **Static Generation**: SEO tags are in HTML (not JavaScript)
- **Fast Indexing**: Crawlers see metadata immediately
- **Rich Results**: JSON-LD enables rich search results
- **Social Sharing**: Open Graph ensures beautiful previews

## Troubleshooting

### Metadata Not Showing
- Clear browser cache
- Rebuild the app (`npm run build`)
- Check that files are in correct directories
- Verify environment variables are set

### Sitemap Not Accessible
- Ensure `app/sitemap.ts` exports a default function
- Check build logs for errors
- Verify it's accessible at `/sitemap.xml` after build

### Icons Not Loading
- Check file paths match manifest.json
- Ensure images are in `/public` directory
- Verify image dimensions match requirements
- Clear browser cache and reload
